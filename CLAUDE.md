## Commands

```bash
# Start all dev services (PHP server, queue, logs, Vite) concurrently
composer run dev

# Run PHP tests (clears config, runs pint lint check, then phpunit)
composer run test

# PHP linting (Laravel Pint)
composer run lint          # fix
composer run lint:check    # check only

# JS/TS linting (ESLint)
npm run lint               # fix
npm run lint:check         # check only

# JS/TS formatting (Prettier)
npm run format             # fix
npm run format:check       # check only

# TypeScript type checking
npm run types:check

# Full CI check (all of the above)
composer run ci:check

# Run a single PHP test file
php artisan test tests/Feature/Tracker/TrackerControllerTest.php

# Run a single PHP test by name
php artisan test --filter test_store_tracker
```

Tests use an in-memory SQLite database — no separate test DB setup required.

## Architecture

This is a **Laravel + Inertia.js + Vue 3** app (the Laravel Vue starter kit). The frontend is a SPA rendered server-side via Inertia, with Vite as the bundler and Tailwind CSS v4 for styling.

### Backend

- **Controllers** live in `app/Http/Controllers/`. Single-action controllers are used where possible (e.g. `DashboardController`).
- **Actions** in `app/Actions/` encapsulate business logic (e.g. `CreateTrackerAction`). Controllers delegate to actions rather than containing logic directly.
- **Form Requests** handle validation (`app/Http/Requests/`).
- **Flash messages** are passed to Inertia via `inertia()->flash([...])` before redirecting. The flash shape is typed in `resources/js/types/global.d.ts` under `flashDataType`.

### Frontend

- **Pages** are in `resources/js/pages/`. Inertia maps controller responses to page components automatically.
- **Layouts** are resolved in `resources/js/app.ts` based on the page name prefix (`auth/` → `AuthLayout`, `settings/` → `AppLayout + SettingsLayout`, default → `AppLayout`). Per-page layout options (e.g. breadcrumbs) are set via `defineOptions({ layout: { breadcrumbs: [...] } })`.
- **UI components** in `resources/js/components/ui/` are built on **Reka UI** primitives (headless, accessible). There is no shadcn-vue dependency — components are hand-built using Reka UI directly.
- **Routes** on the frontend are auto-generated by **Laravel Wayfinder** into `resources/js/routes/` and `resources/js/wayfinder/`. Import route helpers from `@/routes` (e.g. `import { dashboard } from '@/routes'`).
- **Path alias** `@/` maps to `resources/js/`.

### Flash / Toast pattern

Controllers flash a `toast` object via `inertia()->flash()`. The type is `{ type: 'success' | 'error'; message: string }`. Pages read this via `usePage()` (`page.flash.toast`) and pass it to the `<Toast>` component (`resources/js/components/ui/toast/Toast.vue`), which uses Reka UI's `ToastProvider`/`ToastRoot`/`ToastViewport` to display it.
